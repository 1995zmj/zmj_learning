正因为符号之间的所有内容都会被忽略，你有时候会看到程序员对多行注释进行了一定的“美化”：

``` c++
/* 这是一个多行注释。
 * 左对齐的星号
 * 可以让注释更易读。
 */
 #include<iostream>

 using namespace std;
 int main()
 {
     
 }
```

通常，良好的解决方案具有以下特征：

* 它们很简单（不会过于复杂或令人困惑）。
* 它们有很好说明（特别是在提出任何假设或限制时）。
* 它们是模块化构建的，因此可以在以后重复使用或更改，且不会影响程序的其他部分。
* 它们非常稳健，可以在出现意外情况时容易修复或提供有用的错误消息。
